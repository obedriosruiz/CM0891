Contexto de los datos¶

La informacin con la que se cuenta es una base de ventas de una compaa cementera que distribuye distintos tipos de productos a granel y sacos a lo largo de un pas en Suramerica. Dicha informacin contiene todo el nivel de detalle de facturacin y otras caractersticas logsticas y de entrega de las ordenes, incluyendo datos sobre los clientes

1. Recopilación de datos¶

A continuacin se despliega una arquitectura terica para la recopilacin de los datos de la compaa basada en Microsoft Azure

1.1. Arquitectura¶

En este esquema existe una fuente de datos que es poblada por transacciones en lnea ejecutadas en un sistema ERP SAP hacia tablas que son consumidas por transformadores que las llevan hacia un objeto query en BW.

Desde all el query es consultado mediante un query MDX a travs de una actividad copy data en una instancia de Azure DataFactory para cada perodo mensual vencido y desplazado hacia un Azure DataLake en formato .csv con una configuracin de extraccin predeterminada (separadores, encoding y esquema).

2. Detección de características¶

2.1. Caso de negocio¶

A la compaa de cementos le interesara encontrar predicciones certeras de sus ventas en diferentes ventanas temporales, buscando asegurar niveles de confiabilidad de, al menos, $90\%$ en $1$ mes, $80\%$ a $3$ meses y $70\%$ a $6$ meses o ms. Dichas predicciones deben aplicarse a todos los niveles de jerarqua que se tienen en la estructura comercial.

En la medida de lo posible se deseara anexar informacin macro-economica y conocimiento de mercado adicional que permita robustecer las proyecciones; no obstante, esta condicin no es necesaria.

2.2. Interpretación caso de negocio¶

El problema se traduce en encontrar un modelo que consuma la informacin histrica (input) de las ventas de la compaa y genere estimaciones con altos niveles de certeza para la variable de volumen.

2.3. Detección¶

2.3.1. Redundancia de variables¶

En la fuente de informacin, y basndose en el diagrama de jerarquas previamente expuesto, se identifica la oportunidad de remover todos los descriptores clave, entendidos como la representacin en clave de los valores en texto o valor, as, por ejemplo, el campo "Subsegmento" (Edificaciones) tendra un campo gemelo "Subsegmento Clave" (102) el cual se retira por ser redudante. Por ello se procede a la creacin de un diccionario que permita resumir claves y caractersticas y posteriormente del dataset seleccionar los campos clave relevantes.

En cuanto a las variables numricas se conserva exclusivamente la variable "Volumen Real Total (TN - M3)".

def iden_features(df_check, x_cols, y_cols, n_estimators = 1000):
    
    df_check['Mes_natural'] = df_check['Mes_natural'].map(dt.datetime.toordinal)
    
    x = df_check[x_cols].values
    y = df_check[y_cols].values[:,-1]
    
    regressor = RandomForestRegressor(n_estimators = n_estimators, random_state = 0)  
    regressor.fit(x, y)
    
    fi = pd.DataFrame({'feature': x_cols,
                   'importance': regressor.feature_importances_}).\
                    sort_values('importance', ascending = False).reset_index(drop = True)
    
    return fi

La metodologa de seleccin de caractersticas a aplicar consiste en la validacin de importancia de los descriptores para cada muestra por perodo. Para cada perodo $t$ se utiliza un random forest regressor como modelo para estimar la importancia de cada variable y finalmente se seleccionan aquellas que para un corte dado cumplen las condiciones deseadas.

La medida interna de pureza del random forest para el caso de regresin habla sobre la varianza del mismo, as, para cada variable que aumente la pureza sta de considera ms importante.
